# Progress State: 네모로직 코어 엔진 개발 준비

## 1. 현재 진행 상황 (Current Progress)

### 디렉토리 구조 생성
- 프로젝트 최상단에 아래의 4대 핵심 디렉토리를 생성 완료하여 역할을 엄격히 분리하였습니다.
  - `frontend/`: 클라이언트 애플리케이션 (Vue.js / Vite)
  - `backend/`: REST API 서버 (Java / Spring Boot)
  - `infra/`: 인프라 및 프로비저닝 (Terraform / Ansible)
  - `docs/`: 문서 및 개발 진행 상태 기록

### 프론트엔드 환경 초기화 및 TDD 준비
- `frontend/` 디렉토리에 **Vite**를 기반으로 **Vue.js + TypeScript (`vue-ts`)** 프로젝트를 성공적으로 초기화했습니다.
- 테스트 기반 개발(TDD)을 위해 **Vitest** 테스팅 프레임워크를 개발 의존성(`devDependencies`)으로 설치하고 `package.json`에 `test` 스크립트를 추가하였습니다.
- 더미 테스트(`src/dummy.test.ts`)를 작성 및 실행하여 테스트 러너가 오류 없이 작동함을 검증 완료하였습니다.

---

## 2. 다음 단계: 네모로직 코어 엔진 개발 (Next Goals)

코어 게임 엔진은 Vue.js의 반응형 시스템(Ref, Reactive)이나 컴포넌트 생명주기와 완전히 분리된 **순수 스크립트 모듈(TypeScript)**로 작성되어야 합니다. TDD 방법론에 따라, 핵심 비즈니스 로직 구현 전에 반드시 단위 테스트 케이스를 먼저 작성합니다.

### 핵심 작업 목록

1. **테스트 프레임워크 구성 고도화**
   - 코어 엔진 전용 테스트 파일 경로 정의 (`*.spec.ts` 또는 `*.test.ts`)
   - 필요 시 테스트 헬퍼 함수 개발

2. **배열 조작 모듈 (Array Manipulation)**
   - **TDD 진행**: 격자판 생성, 특정 셀 상태 변경(빈 칸, 채움, X 표시)에 대한 테스트 작성 후 구현
   - 격자판 크기(5x5, 10x10, 15x15 등)에 따른 2차원 배열 데이터 구조 모델링

3. **힌트 계산 모듈 (Hint Calculator)**
   - **TDD 진행**: 정답 격자판을 입력받아 가로/세로 힌트 배열을 올바르게 생성하는지에 대한 테스트 작성 후 구현
   - 연속된 채워진 셀의 그룹 길이를 계산하는 알고리즘 구현

4. **검증 모듈 (Validator)**
   - **TDD 진행**: 사용자가 작성 중인 격자판 상태가 정답(혹은 생성된 힌트)과 완벽히 일치하는지 판별하는 테스트 작성 후 구현
   - 게임 클리어 조건 및 오류 상태 검증 로직 작성

---

## 3. 개발 규칙 및 제약사항 준수 확인
- **TDD 필수 준수**: 코어 엔진 모듈 구현 전 테스트 코드가 먼저 작성되고 통과되는 것을 점진적으로 확인합니다.
- **디렉토리 격리**: 이 작업은 오직 `frontend/` 디렉토리 내에서만 수행되며, 타 디렉토리의 파일들과 혼재되어 수정되지 않도록 제어합니다.
- **반응성 디커플링**: 코어 모듈 내에는 Vue 프레임워크 관련 코드를 일절 포함시키지 않습니다.
